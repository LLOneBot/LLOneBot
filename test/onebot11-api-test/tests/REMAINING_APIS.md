## 为什么这些 API 未测试？

### 1. 闪照相关
- 需要特殊的权限和环境
- 闪照功能在某些 QQ 版本中可能不可用
- 测试难度较高

### 2. 群相册相关
- 需要群主或管理员权限
- 相册功能使用频率较低
- 测试环境配置复杂

### 3. 文件高级操作
- 需要先有文件和文件夹
- 操作具有破坏性
- 测试场景复杂

### 4. 系统调试接口
- 属于开发调试功能
- 不建议在生产环境使用
- 可能影响系统稳定性

### 5. 配置和重启
- 会影响整个系统运行
- 测试风险较高
- 需要特殊的测试环境

## 建议

这 17 个未测试的 API 主要是：
- **特殊功能** (闪照、相册)
- **高风险操作** (重启、配置修改)
- **调试接口** (debug、send_pb)

对于常规使用场景，**84.3% 的覆盖率已经足够**，涵盖了所有核心功能和常用扩展功能。

如果需要测试这些特殊 API，建议：
1. 使用独立的测试环境
2. 添加更多的安全检查
3. 使用 `.skip` 标记默认跳过
4. 在文档中明确标注风险
